HTML — Section 2

SEO Meta Tags

Search engines read these tags in the <head> to understand your page. Getting them right improves discoverability.

  • <title>Page title shown in browser tabs and Google search results — the single most important SEO tag
  • meta descriptionShort summary (150–160 chars) shown under the title in search results; affects click-through rate
  • meta robotsControls whether search engines index the page and follow links (index, noindex, follow, nofollow)
  • meta viewportEssential for mobile — tells browsers how to scale the page; Google penalizes non-mobile-friendly pages
  • meta keywordsLargely ignored by modern search engines but still used as documentation for content intent
  • og:title / og:descriptionOpen Graph tags control how your page appears when shared on social media (Facebook, LinkedIn, Slack)
  • canonical linkPrevents duplicate content penalties by telling search engines which URL is the "real" version
  • lang attributeSetting lang="en" on <html> helps search engines serve the page to the right audience

CSS — Section 5

6 Font Sizes — Inline Styling

Same text, six different size units. A mix of absolute (px, pt) and relative (em, rem, %, vw) units — each explained with a real-world use case.

  • The quick brown fox jumps over the lazy dog. 12px — Absolute. Good for fine print, footnotes, or legal disclaimers where size must never change regardless of user settings.
  • The quick brown fox jumps over the lazy dog. 1rem — Relative to root font size. Best for body text — respects the user's browser font preference (e.g., if they set 20px as default, this scales up too).
  • The quick brown fox jumps over the lazy dog. 1.25em — Relative to parent element font size. Useful inside components like cards or blockquotes where you want a heading slightly larger than the card body.
  • The quick brown fox jumps over the lazy dog. 120% — Relative to parent. Similar to em but can feel more intuitive when working with designers who think in percentages.
  • The quick brown fox jumps over the lazy dog. 18pt — Absolute (1pt = 1/72 inch). Most useful for print stylesheets (@media print) where physical size matters.
  • The quick brown fox jumps over the lazy dog. 3vw — Relative to viewport width. Great for hero headings that should scale with the screen size, like large banners or full-page titles.

CSS — Section 6

4 Positioning Options

CSS gives you fine-grained control over where elements appear. Below is a live demo with all four main types.

static — default flow
relative — nudged 12px down, 20px right
sticky — stays at top: 80px while scrolling
absolute — anchored to bottom-right corner
  • static — the default. Element sits in the normal document flow. Use when you don't need special placement.
  • relative — element stays in flow but can be nudged with top/left/right/bottom. Great for fine-tuning an element's position without affecting neighbors.
  • sticky — acts like relative until you scroll past it, then sticks to a position. Perfect for navbars and section headings that should follow the user down the page.
  • absolute — removed from the document flow and positioned relative to the nearest positioned ancestor. Use for tooltips, badges, and overlays like "New" labels on cards.
  • fixed — like absolute but relative to the viewport, not an ancestor. The footer of this page uses position: fixed to always stay at the bottom of the screen.
